# ✨ Portfolio
自身の経歴、習得技術、制作物がわかるようポートフォリオを作成した。  
ポートフォリオを閲覧することで、経歴、内面、制作物一覧等がわかるものとした。  
ポートフォリオの「Projects」画面には、ハッカソン（チーム開発）、個人開発で作成した制作物が複数あり、クリックするとGitHubのリポジトリに遷移する。

## 背景
企業の採用担当者は、時間が限られている中で、自社に合う人材か否かを見極める必要があるとともに、応募者側が制作物のGitHubリンクを企業へ公開するだけでは、企業側の確認・判断のコストが大きいと考えた。  
よって、企業側の確認・判断のコストを下げるとともに、自社に合う人材か否か迅速に判断できる一助となるよう、自身の「内面、スキル、制作物」等を集約したポートフォリオを公開することとした。  

## ターゲット
- 企業の採用担当者

## 使用技術
本ポートフォリオは静的ファイルで構成されており、カスタムドメインを取得し、Web上に公開している。  
よって、主要使用技術のカテゴリは、「フロントエンド」および「インフラ」である。  
以下のとおり、使用技術を示す。  

| **カテゴリ** | **技術** |
| --- | --- |
| フロントエンド | React 19.2.0 / Next.js(APP Router) 16.0.7 / Tailwindcss / TypeScript |
| インフラ | Vercel |
| その他 | Canva / GitHub |

## 技術選定理由
高速レスポンスによるユーザービリティの向上を図るため、SSGを有する**Next.js**を採用した。  
また、Next.jsとVercelを使用することで、高速デプロイと自動ビルド・デプロイを可能とし、ポートフォリオの制作から公開までの期間の短縮を図るとともに保守性・運用性の向上を図った。  
本ポートフォリオは、DBと連携した複雑な処理はないが、自身の学習目的のため、**Typescript**を使用した。  

## 画面設計
### デザインの方向性
- 余白を大きく取り、文字数を極力抑えることで、スキルやプロジェクト等の見せたいのものを目立たせる。
- 目に優しく、文字が見やすくなるよう、色数は2～3色までに抑える。
- プロフィール画像、プロジェクト画像を目立たせたいため、できるだけ無彩色（黒、グレー、白）を使用する。

### 各画面の役割
**1) Home**

スタート画面であり、閲覧者に興味をもってもらうために、動きをつけつつ文字数を極力抑える。  
習得技術、各種リンクを掲載し、必要となる情報・ソースへ誘導する。  

**2) About**

自身の経歴、ITエンジニアへ転職する理由、学習状況を簡潔に伝える。  
どの程度のスキルがあるかをスキルレーダーチャートを用いて、直感的にわかるよう伝える。  

**3) Projects**

ハッカソン（チーム開発）、個人開発での制作物を集約する。  
制作物にはアプリの画像、アプリの概要、自身の担当等を記載し、アプリの全体像を掴みやすくする。  
プロジェクトカードをクリックすると、自身の制作物のGitHubリンクへ誘導するよう設定する。

## 開発環境構築手順
### 1) 本ソースのclone
各自のローカル作業ディレクトリに移動した後に、本ソースをcloneする。  
注）作業ディレクトリに日本語が含まれていたり、階層が深い場合には、後述する「npm run dev」でエラーとなる可能性があるため、極力日本語を避けた比較的浅い階層でのディレクトリで作業することを推奨する。  

```
git clone https://github.com/hiro-53-RareTECH/Portfolio.git
```
もしくは
```
git clone git@github.com:hiro-53-RareTECH/Portfolio.git
```

### 2) 対象ディレクトリへ移動、依存ライブラリのインストール
クローン後に「Portfolio」というディレクトリが作成されるため、そこへ移動する。
```
cd Portfolio
```

移動後に依存ライブラリのインストールを行う。
```
npm install
```

### 3) ローカル環境での立ち上げ
以下のコマンドで、ローカル環境を立ち上げる。  
```
npm run dev
```

デフォルトでは、以下のとおり、3000番ポートが解放されてlocalhost上に表示されるため、ブラウザのURLに入力して確認する。  
注）ポート番号はコンフリクトを避けるため、既に3000ポートが使用されている場合は、別のポート番号が割り当てられる。その際はnpm run dev後のコンソール画面を確認し、「Local」および「Network」を確認する。  
```
http://localhost:3000
```
